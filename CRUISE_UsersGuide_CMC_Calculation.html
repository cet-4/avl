CMC Export

The Common Model Compiler (CMC) feature packs model input data with CRUISE Kernel into an executable or library. Such binaries, which can be standalone .exe, shared library .dll or static library can be used to simulate only the model from which it was created.

To prepare the model for CMC export add CRUISE Interface component, or one of it's derivatives: ETAS, dSPACE, NI Veristand, OPAL_RT, to the model and define input and output databus channels. Databus channels are used for data exchange between CMC exported CRUISE model and the external application that uses produced .dll. Input channels are used for sending data to external application, and output channels on CRUISE Interface are used for passing data from external application to CRUISE model.

Note:
User should take care when specifying units on CRUISE Interface databus channels. Values that come at input channels will be converted to specified units before sending them to external application. Units specified (Interon output channels just denote in which units values are coming from external application.

After the model is prepared for CMC, proceed to the Calculation Center where new executables with built-in data will be created. After adjusting some settings, the CMC is started by clicking on - Start calculation.

The CMC executable or library can be created for Windows and dSPACE platforms. Depending on the selected option, the resulting output is 
<model_name>.exe
, 
<model_name>.lib
, 
<model_name>.dll
(Windows). Output files will be located in the '
<project_path>\inc\cmc\cmc_output
' folder.

Once created, the CMC executable or library can be copied where needed and used in the same way as the CRUISE Kernel would be used. A significant difference is that the calculation is limited to the model which was used to create the CMC executable or library.

The only way to alter certain specific input data is through favorites xml files. It is important to create and save the xml file before starting to create the CMC executable or library. The appropriate license is required to change certain input data using xml files while running the CMC executable.

To run 
<model_name>.exe
with new data, place the xml file in the same directory. Also, if the model uses any additional dynamic library, it is important that it is placed in the same directory as the CMC executable/library or the current working directory.

There are two main categories of CMC executables and libraries:

  1. without output

    This is useful if CMC will be used on the real time platforms which do not have the usual input/output capabilities.

  2. with output

    This is useful when allowing the results of a certain model to be viewed without viewing the model itself and/or changing input data.
