Channel Numbers

The channel numbers are used in the template files to make a correlation to the input, respectively result data. In its simplest form, a channel input will have the following format: 
#12345600@

The Channel Wizard or the Data Handbook (also available in HTML format) are available to create or modify a template file and to find the numbers of certain channels.

Immediately in front of @, other formatting options, which allow a special representation of the output values, can be stated.

1 Sample number means the number, e.g.: of one wheel, in the vehicle. With the sample number a special wheel can be described and the results of one wheel can be shown.

The formalization also allows the combination of several formatting instructions:

e.g. #1015000y14%4.2[ km/h ]fls"1st Gear"@

The formatting instruction described above has the following results:

The result values of the 14th column (y14) of the transferred Table of values (Channel Number: 10150000) are to be represented with a floating point with 4 places in front of the point and 2 places behind the point (%4.2).

Furthermore, the values are to be converted from the SI value [m/s] to [km/h].

The unit is to be stated in front of the values (f), and the values are to be leftFlush (l). (leftFlush will automatically replace the statement of the places in front of the point)

The additional text ("1st Gear:") also is to be stated in front of the unit (s) in each line of the column in the output file.

The formatting instruction has to be finished with the @ Character.

The result would look as follows:

1st Gear:  [km/h]  17.45
1st Gear:  [km/h]  21.33
1st Gear:  [km/h]  26.89
...
  • Sample Results

    The last two numbers of a channel number refer to the samples, i.e. if there are four wheels in a model the corresponding channel has four samples. If the channel number is defined with #xxxxxx00@, all samples are printed out (one column), with #xxxxxx01@ one special sample result (1st sample) can be printed.

    Example: (static rolling radius in the component wheel; equal for all four wheels)

    #11400800w%5.3@                               0.255
                                                  0.255
                                                  0.255
                                                  0.255

    respectively:

    #11400801w%5.3@                               0.255
  • Diagrams with Sample Results

    With the formatting instruction "i" a list with two columns can be defined, which allows a diagram representation for different sample results by using the PPD- diagram generator. In the first column the sample number is located, in the second one the corresponding sample results.

    Example:

    #11400800iw%5.3@                        1          0.255
                                            2          0.255
                                            3          0.255
                                            4          0.255
  • Variation Diagrams vx

    With the formatting instruction "vx" a list can be defined which contains all variation results. Only results of the variation calculation are listed, the results of the standard calculation are not shown. All variations are saved in a variation file.

    Example: (e.g.: 5 Variations of the vehicle mass corresponding consumption results)

    #10150100v1w%5.1@#30150100vw%5.2[l/100km]@

    leads to: 

    1000.0                   8.42
    1200.0                   9.54
    1400.0                  10.32
    1600.0                  11.55
    1800.0                  12.47
  • Conditional Output

    A block standing between the brackets ^{ and ^} is not shown in the results file if no one of the channel numbers inside has a valid value. The brackets can stand at the beginning of a line (like it is shown below) but they can also be located somewhere in the line. 

    #10150100vw%5.1@
    ^{
    Optional Result:
    #30150100@
    #30150200@
    ^}
  • Definition of special Output files

    A block marked with the brackets "^[" and "^]" is not written into the file 'results.log' but into a special file. The brackets have to be stated always at the beginning of a new line. These lines are automatically deleted during the result output.

    After the bracket "^[" the name for the special file can be defined (marked with ""). If the filename is defined without extension automatically the extension ".log" is added. If no filename is defined the marked block is written in the file "summary.log" .

    Example: The file

    Mass:     #10150100%5.1@
    ^[
    Mass:     #10150100%5.1w@
    ^]
    Speed:    #10405500y1x1[1/min.]%5.1@
    ^["test.txt"
    Mass:     #10150100%5.1@
    ^]
    ...

    leads to the following outputs:

    Results.log:

    Mass:      1000.0 <kg>
    Speed:      600.0 <1/min>

    summary.log:

    Mass:      1000.0

    test.txt:

    Mass:      1000.0 <kg>
  • String Output

    s...... String stated in front of the value

    s4.... String stated in front of the value until the 4th character, the rest is stated behind the value

    Example:

    #10405500y1%5.1[1/min]s4"str1str2"@

    leads to 

    str1   600.0 <1/min>  str2
    str1   800.0 <1/min>  str2
    str1  1000.0 <1/min>  str2
  • Coupling of Channels

    Single Channels can be coupled by operators.

    The coupling is made in the following way:

    #<<Channel1,Channel2,OP1,Channel3,OP2,Channel4,OP3,..>>Formatting Instructions@

    The channels can be single values as well as single columns or lines of a table. It is also possible to define constant values instead of a channel; but the first argument has to be a channel. At the definition of the channels please check for the unit. Eventually a conversion into another unit is needed at the definition of the channels. During calculation, all channel values are transferred to basic units. For constant values, the basic unit of the previous channel (that means, of the previous argument) is used (not the eventually explicitly defined unit).

    OPx represents the operator, where the following are available:

    ADD (2)
    ..............for addition
    SUB (2)
    ..............for subtraction
    MUL (2)
    ..............for multiplication
    DIV (2)
    ...............for division
    SEARCH_Y (3)
    ....arguments: list 1, value, list 2. The result is the list of these values of list 2, whose corresponding list 1-values are equal to the argument value. If there is no equation, the results are interpolated / extrapolated.

    If a continuous series of values equals to the argument value, only the last value of this series is used. 

    SEARCH_X (3)
    ...arguments: list 1, value, list 2. The result is the list of indices of these values, which would be the results of the SEARCH_Y-function (by using the same arguments)
    MAX (1)
    .............for the maximum of a list
    MIN (1)
    .............for the minimum of a list
    MAX_X (2)
    .........for the search for the value of the first list which corresponds to the first maximum of the second list
    MIN_X (2)
    ..........for the search for the value of the first list which corresponds to the first minimum of the second list
    SIN (1)
    ..............for the sinus values of a list
    COS (1)
    .............for the cosinus values of a list
    TAN (1)
    ..............for the tangens values of a list
    ASIN (1)
    ............for the arcsin values of a list
    ACOS (1)
    ...........for the arccos values of a list
    ATAN (1)
    ............for the arctan values of a list
    EXP (1)
    .............for the exponential values of a list
    LN (1)
    ...............for the ln values of a list
    LOG (1)
    .............for the log values of a list
    POW (2)
    ...........arguments: list, value. The results are the list values with the power of the value
    SQRT (1)
    ...........for the square root values of a list
    ABS (1)
    .............for the absolute values of a list

    Some examples of this:

    For all examples the following channels are used:

    #10403200y1@             with the values      243.15
                                                  293.15
                                                  377.15
                                                  378.15
                                                  473.15
    #10403200y2@             with the values        0.01
                                                  210.0
                                                  210.0
                                                 2100.0
                                                 2100.0
    #10403200y1x3@         with the value         377.15
    #10403200y2x3@         with the value         210.0
    #(1)<<#10403200y1@,#10403200y2@,OP>>%8.2w@

    couples the columns #10403200y1@ and #10403200y2@. The single values of the columns are coupled with each other. The results can be stated with a "usual" Short-cut number (in this case #*1@). This calculation leads to the following results for the different operators:

    ADD               SUB               MUL                DIV
     243.16         243.14              2.43            24315.00
     503.15          83.15          61561.50                1.40
     587.15         167.15          79201.50                1.80
    2478.15       -1721.85         794115.00                0.18
    2573.15       -1626.85         993615.00                0.23

    The number of values of the coupled columns must exactly be the same.

    If one column is coupled to a constant value, the coupling is performed for the whole column. E.g. the definition 

    #(3)<<#10403200y1@,2,OP>>%5.2w@

    leads to the following results:

       ADD            SUB                MUL            DIV
     245.15         241.15             486.30          121.57
     295.15         291.15             586.30          146.57
     379.15         375.15             754.30          188.57
     380.15         376.15             756.30          189.07
     475.15         471.15             946.30          236.57

    Lists of the length 1 are calculated like constant values.

    #(4)<<#10403200y1x3@,#10403200y2x3@,OP>>%5.2w@

    couples the single values of both tables. This leads to the following results:

       ADD           SUB                MUL             DIV
     587.15        167.15            79201.50           1.80

    Of course also a combination of more than one inputs is possible. E.g. the definition 

    #(5)<<#10403200y1@,#10403200y2@,ADD,2,MUL,2,DIV,#10403200y2@,SUB>>%5.2w@

    delivers as results the first column again

     243.15
     293.15
     377.15
     378.15
     473.15

    The unit of the output is not defined by default. If the unit should is also stated a question mark is stated instead of the unit.
