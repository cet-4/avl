AVL DRIVE Interface

The component AVL DRIVE enables an evaluation of the drivability of CRUISE results. This evaluation is possible only in simulation mode and therefore only in the tasks which provide this mode.

Before starting to work with this interface for the first time, three things must be done:

  1. A DRIVE installation with a valid license must exist.
  2. The file 'Cruise.cdl' must be copied into the configuration directory of AVL DRIVE, which may be created on a different location than the AVL DRIVE installation directory; therefore copy the 'Cruise.cdl' file from 'config' folder of the CRUISE installation directory to the following folder of AVL configuration directory: '
    .\program\DCCSGR\ImportTemplates\
    ', e.g. into '
    C:\AVLDrive\program\DCCSGR\ImportTemplates\
    '. This file contains the interface definition of the channels.

  3. In the local environment of CRUISE (Tools -> extras -> local environment) two variables have to be set as follows:

    • DRIVE_START='AVLDrive'
    • CRUISE-CDL-FILE='Cruise'
  4. In the system environment variables path to the 'Program' folder of AVL DRIVE installation directory has to be set. Therefore select 'Path' variable in the system environment (accessible from CRUISE: Tools -> extras -> local environment) and add before mentioned path e.g. '
    C:\AVL\AVLDRIVE_V3.4.0\Program
    '.

Before evaluation start, a model must be parametrized also in DRIVE. Therefore the program is launched by pressing the 'Edit with AVL DRIVE' button, then select the required drive type and specify the important parameter e.g. with the help of the Wizard.

IMPORTANT: Do not use space characters or special characters for the DRIVE model name. This could cause problems with an automatic call of the model in future.

Then the vehicle-setup '*.vsf' must be saved.

Check the parameter again by choosing the menu item 'Vehicle' in the DRIVE main window and then the option 'Edit Work Parameter'.
