dSPACE

For the dSpace CMC calculation a license is required.

dSPACE tools with GNU C Compiler are installed in addition to the Windows VisualC platform requirements for using dSPACE builder from CMC. The end product of the calculation is a library which can be linked with Cruise S-function in MATLAB/Simulink during the build process for dSPACE DS1006 RT system. The library is created in two steps:

  1. CMC is called and CMC .c files of a current model are created. These files contain all the input data of a selected model.
  2. CMC .c files are compiled with dSPACE compiler and linked with Cruise library for dSPACE that comes with CRUISE installation.

If the default settings are not satisfactory, adjust the settings on the CMC page before starting to build a new dSPACE library of the current model.

It is possible to create CMC executable or library for Windows, Linux and dSpace platform. Depending on selected CMC-Mode options, the resulting output is <model_name>.exe, lib<model_name>.lib, <model_name>.dll (Windows) or <model_name>.exe, lib<model_name>.a, <model_name>.so (Linux). Output files will be located in the '<project_path>\inc\cmc\cmc_output' folder. Once created, CMC executable or library can be copied where needed and used in the same way Cruise Kernel would be used. Significant difference is that the calculation is limited to the model which was used to create CMC executable or library. The only way to alter certain specific input data is through xml files, but this is possible only if it was anticipated during the creation of the CMC executable or library. There are two main categories of CMC executables and libraries:
  • without output;
  • with output;

Without output version is very useful if CMC will be used on the real time platforms which don't have the usual input capabilities. With output version is useful when you want to give somebody to view the results of a certain model, and you don't want to enable viewing the model itself and/or changing of input data.

CMC exe:

This file can be run on any PC, whether it has a Cruise installation or not, by double click. When exe is started, command prompt window is opened where user can monitor calculation progress. If the exe file was built using with output mode, output files are created. Files that are created are the ones that were selected before starting CMC of a selected model. Running exe will always have the same outcome (if no xml file is available). Model that is calculated will be a model that was opened in Cruise when CMC was started, and results will be the ones that Cruise would have calculated for this specific model.

CMC lib:

It has same usage capabilities as CMC.exe, but user has to provide a main program that will be linked with this library. Main program has to call appropriate functions from library in appropriate order to get a good results. This is mostly used in Matlab to link S-function with Cruise.

CMC dll:

Same capabilities as CMC lib, only this time, linking of main program (i.e. S-function) is done dynamically and not statically like with lib.

All the setting descriptions are Windows VisualC related. LoadLib option is off by default because only static link is supported by dSPACE. CMC-Mode is set to without output by default because no file manipulation functions are available on dSPACE. Three additional options are available:

  • Task
  • Folder
  • MATLAB
  • Online Parameter Access

Task and Folder are necessary for running CRUISE from S-function, because only one task and one folder can be run when CRUISE is used externally. The MATLAB checkbox is used to create dummy model for dSPACE. When activated MATLAB is started and CreateModel function is executed. Online Parameter Access is used when there are parameters defined in the model. For more information refer to the CRUISE Interfaces.
